Пример разбора: создаём адаптивную карточку (Flex + Grid)

🎯 Задача: карточка товара, которая меняет сетку на разных экранах

Условие: Нужно сверстать секцию из 3 карточек. На десктопе — 3 колонки, на планшете — 2, на телефоне — 1 колонка. Внутри карточки: изображение, заголовок, цена и кнопка (Flex для выравнивания).

Алгоритм (по шагам):

  1. Создаём HTML-структуру: <div class="products-grid"> с тремя карточками.
  2. Для родителя используем display: grid и grid-template-columns: repeat(3, 1fr).
  3. Добавляем gap: 1.5rem для отступов между карточками.
  4. Внутри карточки: display: flex; flex-direction: column; чтобы кнопка прижималась к низу.
  5. Пишем медиазапросы: @media (max-width: 768px) → 2 колонки, @media (max-width: 480px) → 1 колонка.
<!-- HTML -->
<div class="products-grid">
    <div class="product-card">
        <img src="image.jpg" alt="товар">
        <h3>Стильный свитшот</h3>
        <div class="card-footer">
            <span>49 BYN</span>
            <button>Купить</button>
        </div>
    </div>
    // ещё 2 карточки ...
</div>

<!-- CSS -->
.products-grid {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 1.5rem;
}
.product-card {
    display: flex;
    flex-direction: column;
    background: white;
    border-radius: 1.5rem;
    overflow: hidden;
}
.card-footer {
    display: flex;
    justify-content: space-between;
    margin-top: auto;
}
@media (max-width: 768px) {
    .products-grid { grid-template-columns: repeat(2, 1fr); }
}
@media (max-width: 480px) {
    .products-grid { grid-template-columns: 1fr; }
}

🧪 Пример упражнения: анализ кода (понимание циклов и условий)

Часто вёрстка пересекается с базовой логикой. Разберём задачу на понимание работы цикла (полезно для анимаций и динамических классов).

let s = 0;
for (let i = 1; i <= 10; i++) {
    if (i % 2 === 0) {
        s += 1;
    }
}
console.log(s);

Разбор: Цикл от 1 до 10, условие проверяет чётность. Чётные числа: 2,4,6,8,10 → срабатывает 5 раз. Значение s = 5. В пособии подобные задания помогают легче понимать работу JavaScript-событий и условных рендеров.
